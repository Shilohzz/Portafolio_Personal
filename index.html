Sobre mí

Diseño sistemas
que no se caen.

Soy Juan Bareño, desarrollador Java Back-End júnior enfocado en construir sistemas bien estructurados, seguros y listos para escalar. Mi especialidad es el ecosistema Spring Boot: desde el diseño del modelo hasta la autenticación y la documentación de la API.

Lo que me diferencia: no solo escribo código que funciona — escribo código que cualquier developer puede entender, mantener y extender. Arquitecturas en capas bien definidas, DTOs, manejo de excepciones limpio y Swagger para que la integración nunca sea un problema.

En LogiTrack reemplacé las hojas de Excel de una empresa real con una API REST completa con JWT. Eso es lo que busco: resolver problemas reales con software real.

LogiTrack — Sistema de Gestión de Bodegas

Sistema full-stack para LogiTrack S.A., empresa de ejemplo para el proyecto, que administra bodegas en distintas ciudades de Colombia. Reemplazó por completo la gestión en hojas de Excel con una API REST segura, documentada y con frontend integrado.

  • CRUD completo de bodegas, productos, clientes, categorías y usuarios
  • Registro de movimientos: entradas, salidas y transferencias entre bodegas
  • Autenticación y autorización con Spring Security + JWT
  • Documentación interactiva de endpoints con Swagger / springdoc-openapi
  • Frontend HTML/CSS/JS servido desde el mismo proyecto Spring Boot
  • Arquitectura limpia: controller → service → repository → model
  • Manejo global de excepciones y DTOs con mappers dedicados

TECNO_STORE — Sistema de Gestión de Ventas

Sistema de consola para tienda de tecnología bajo patrón MVC y persistencia DAO. Centraliza inventario, clientes, empleados y ventas con integridad transaccional a nivel de base de datos.

  • Patrón MVC con separación clara de responsabilidades
  • Arquitectura de persistencia DAO para acceso a datos
  • Validación de stock en inventario antes de procesar cada venta
  • Transacciones SQL para garantizar integridad referencial
  • Vinculación de ventas con clientes y empleados registrados
